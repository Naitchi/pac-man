## Development Progress

### Week 1 - Project Foundation

The first week focused on establishing the technical foundation of the project:

* Repository structure
* Dependency management with UV
* Makefile
* A-Maze-ing package integration
* Configuration parser
* Highscore management

The goal of this phase was to ensure that all mandatory technical requirements were working before gameplay development began.

### Week 2 - User Interface and Core Gameplay

The second week focused on the graphical interface and gameplay systems:

* Pygame experimentation
* Display system
* Main menu
* Maze rendering
* Entity management
* Player movement
* Collision handling
* HUD implementation

At the end of this phase, the game was already playable and the main systems were connected together.

### Week 3 - Gameplay Completion and Delivery

The final week focused on completing the game and preparing the final delivery:

* Ghost behaviors
* Scoring system
* Level progression
* Cheat mode
* Packaging and executable generation
* Itch.io deployment
* Testing and bug fixing
* Documentation

This phase was dedicated to polishing the project and ensuring compliance with all subject requirements.

## Schedule Analysis

The project followed the planned development order without major blockers.

The technical foundation was completed early, which allowed gameplay and interface features to be developed on top of a stable architecture.

Regular communication and task sharing helped avoid duplicated work and allowed both contributors to work on different parts of the project simultaneously.
